# PrepGrid: AI Interview & Practice Platform (PS1) - Roadmap & Work Distribution

## Project Overview
A complete placement preparation platform helping engineering students with mock tests, coding practice, and AI-driven feedback.

---

## 🟢 SHARED FOUNDATION (Do this first - 30 mins)
Agree on the `User` and `Question` schemas to ensure data consistency.
*   **User Model**: Needs `streak`, `solvedQuestions[]`, `quizScores[]`, `isPro`, and `billingHistory[]`.
*   **Question Model**: Needs `title`, `description`, `testCases[]`, `difficulty`, `topic`.

---

## 🔵 TRACK A: PRACTICE & QUIZ SPECIALIST
**Goal**: Build the coding engine, the dynamic quiz system, and the management console.

### 1. Coding Practice Module
*   **Frontend**: Create `client/src/features/practice/`. Implement the Code Editor (Monaco/CodeMirror) and topic filters.
*   **Backend**: Create `server/src/routes/practiceRoutes.js`.
*   **Integration**: Connect to **Judge0 API** for real-time code execution.
*   **Feature**: Status tracking (Solved/Unsolved) and bookmarking.

### 2. AI Quiz Module
*   **Frontend**: Create `client/src/features/quiz/`. Build the timed MCQ interface.
*   **Backend**: Create `server/src/routes/quizRoutes.js`.
*   **AI**: Prompt Gemini to generate 10 unique MCQs + explanations on a given topic (e.g., "DBMS Normalization").
*   **Feature**: Leaderboard API to fetch top scorers per topic.

### 3. Admin Panel
*   **Feature**: Build a `/admin` route to perform CRUD operations on the Coding Question bank.
*   **Feature**: View all registered users and their activity logs.

---

## 🔴 TRACK B: PLATFORM & INTELLIGENCE SPECIALIST
**Goal**: Enhance the mock interview experience, build the analytics dashboard, and monetize.

### 1. Advanced AI Interview (Upgrading Unfumble.AI)
*   **Adaptive Difficulty**: Refine `interviewMachine.js` so that if a user answers 2 questions correctly, the difficulty jumps from EASY to MEDIUM.
*   **Role Selection**: Allow users to choose specific paths (Frontend, Backend, DSA) instead of just "Resume-based".
*   **History**: Build the "Session History" view so users can review past AI feedback and transcripts.

### 2. Pro Dashboard & Gamification
*   **Analytics**: Build the dashboard using `recharts` to show:
    *   Radar chart of "Weak Areas" based on AI evaluation scores.
    *   Line chart of "Average Score" over time.
*   **Streak Tracker**: Logic to increment a `streak` counter if the user performs 1 action (Interview/Practice) per day.
*   **Status**: Display total questions solved and interviews attempted.

### 3. Payments & Tiers (Monetization)
*   **Tiers**: Logic to limit "Free" users to 5 interviews and 10 practice questions.
*   **Integration**: Setup **Razorpay Sandbox**. Create a "Pro Tier" checkout flow.
*   **Billing**: Build a "Billing History" page showing past transactions.

---

## 🛠️ INDEPENDENT WORK PROTOCOL
1.  **Git Isolation**: 
    *   Track A: Branch `dev/practice-quiz`
    *   Track B: Branch `dev/platform-ux`
2.  **API Isolation**: 
    *   Track A owns: `/api/practice`, `/api/quiz`, `/api/admin`
    *   Track B owns: `/api/interview`, `/api/user`, `/api/payments`
3.  **Database**: Both can use the same MongoDB cluster but should never delete fields the other person added.

---

## 📅 MILESTONES
*   **Milestone 1**: Core Practice (Judge0) + Role-based AI Interview.
*   **Milestone 2**: AI MCQ Generator + Pro Dashboard (Analytics).
*   **Milestone 3**: Payment Integration + Admin Panel + Final UI Polish.
*   **Integration Day**: Merge branches, test full flow, and prepare the demo video.
